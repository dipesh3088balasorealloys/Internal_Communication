Executive Summary

BAL Connect is a fully self-hosted internal communication platform designed to replace Microsoft Teams for 300+ employees at Balasore Alloys Ltd. The platform provides real-time messaging, audio/video calling via Pure WebRTC (browser-to-browser), email integration, calendar scheduling, file sharing, and presence management. All services run on the company's internal network with zero external dependency and zero recurring software licensing cost.

Key Change (v2.0): SIP/UCM6304 server removed from architecture. All calling handled via Pure WebRTC with Socket.IO signaling. This eliminates SIP hardware dependency and simplifies deployment.

WebRTC Calling Architecture

📞 How Calls Work (Pure WebRTC)
1User A clicks call button → browser captures audio/video via getUserMedia()
2Socket.IO sends call:initiate event to server with SDP offer
3Server relays offer to User B via their personal Socket.IO room
4User B sees incoming call modal, clicks Accept → sends SDP answer back
5ICE candidates exchanged via Socket.IO for NAT traversal
6Direct peer-to-peer audio/video stream established between browsers
7coturn TURN server used only when direct connection fails (different subnets)
✅ Why Pure WebRTC (No SIP)
Simpler ArchitectureNo SIP server hardware dependency
Zero Hardware CostNo PBX/gateway device needed
Browser NativeWebRTC is built into all modern browsers
Lower LatencyDirect peer-to-peer, no SIP proxy hop
Easier MaintenanceSoftware-only, no firmware updates
ScalableAdd Janus/mediasoup later for 40+ group calls
Note: UCM6304 SIP server was removed from architecture (firmware failure). SIP.js code retained as backup for potential future use.

☐Clone repository to server
