🔄 Key Data Flows

💬 Real-Time Messaging

1
User types message in MessageInput component
2
Socket.IO emits message:send event to server
3
Server saves to PostgreSQL messages table + allocates sequence number
4
Redis pub/sub broadcasts to conversation room
5
All participants receive message:new event, UI updates instantly

📹 Video Call (Pure WebRTC)

1
Caller clicks call button, getUserMedia(1080p) captures camera
2
Socket.IO sends call:initiate with SDP offer to server
3
Server relays to callee via user:${userId} Socket.IO room
4
Callee accepts, sends SDP answer + ICE candidates exchanged
5
Direct peer-to-peer media stream established (coturn TURN if needed)

🟢 Presence & Status

1
User connects, client emits presence:update (online)
2
Server updates Redis presence key (5-min TTL) + PostgreSQL
3
presence:changed broadcast to all connected users
4
Client heartbeat every 30 seconds refreshes TTL
5
On disconnect, Redis TTL expires, status set to offline

📅 Calendar Event

1
User creates event in CreateEventModal with attendees
2
POST /api/calendar/events saves to PostgreSQL (transaction)
3
event_attendees rows created (creator = accepted, others = pending)
4
Socket.IO emits calendar:invitation to each attendee
5
Attendees see invite, can Accept / Decline / Tentative
