Security Architecture

🔒 Application Security

• JWT authentication (15min access + 7d refresh tokens)
• bcrypt-12 password hashing
• Role-based access control (Admin/Employee)
• Helmet.js security headers
• CORS restricted to internal domain
• Input validation on all endpoints
• File upload: blocked executables, 100MB limit

🛡 Infrastructure Security

• Internal network only (192.168.x.x)
• HTTPS/TLS encryption (self-signed SSL)
• Docker container isolation
• PostgreSQL + Redis: localhost only
• FortiGate firewall: VIP + NAT rules
• Only port 25 (inbound) + 80/443 exposed
• WebRTC: DTLS-SRTP encrypted media

✉ Email Security

• SPF: v=spf1 ip4:192.168.10.15 include:spf.protection.outlook.com
• DKIM: ED25519 + RSA dual signing
• DMARC: p=quarantine/reject policy
• STARTTLS encryption in transit
• Not an open relay (verified by mxtoolbox)
• Per-user SMTP/IMAP authentication

📊 Audit & Compliance

• Admin compliance search across all messages
• Chat export: CSV + TXT transcript
• Message edit/delete tracking (is_edited flag)
• User activity logging
• All data on-premise: zero cloud dependency
• ZERO data leakage — nothing leaves internal network
