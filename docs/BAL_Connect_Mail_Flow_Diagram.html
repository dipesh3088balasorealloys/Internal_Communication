Why Direct Port 25 Outbound Doesn't Work
1

Stalwart tries to send directly to MS 365 on port 25

192.168.10.15 → NAT → 117.240.74.196 → Internet → 52.101.144.3:25 (MS 365)

✗

ISP (BSNL) Blocks Outbound Port 25

Root cause: BSNL (Bharat Sanchar Nigam Ltd) blocks ALL outbound port 25 traffic from their network to prevent spam.
This is an ISP-level restriction — cannot be fixed by firewall configuration.
Proof: TCP connects but SMTP greeting never received (ETIMEDOUT on all external SMTP port 25 targets).
Port 587 works: ISPs don't block port 587 (submission port) — only port 25.

❌ ISP BLOCKS

This is why we use Office 365 SMTP relay on port 587 instead of direct delivery on port 25.
Port 587 is the standard submission port — ISPs allow it because it requires authentication (preventing spam).
Firewall outbound policy (Policy ID 54) is correctly configured — this is NOT a firewall issue.

★
