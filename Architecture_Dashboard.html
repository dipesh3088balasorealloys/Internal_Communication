Internal Communication Platform

Complete MS Teams Replacement — Self-Hosted, Secure, Zero Cost
✓ 300+ Users ⚙ Internal Network Only ★ Grandstream UCM6304 (192.168.7.2) $0 Licensing Cost
🎯 Project Objective

Build a fully self-hosted internal communication platform to replace Microsoft Teams for 300+ employees. The platform provides real-time messaging, audio/video calling (1:1 and group), file sharing, screen sharing, and presence management — all confined to the company's internal network. Zero external dependency. Zero licensing cost.

Key Constraint: Fully utilize the existing Grandstream SIP server. Zero additional software licensing cost. One blank server provided for deployment.

Security Design

🔒 Network Security
  • ✅ All services bind to internal IP addresses only
  • ✅ Firewall blocks all inbound/outbound external traffic
  • ✅ VLANs separate comm server from general network
  • ✅ All traffic encrypted via TLS (internal CA)
🔐 Application Security
  • ✅ JWT-based authentication with refresh tokens
  • ✅ Password hashing with bcrypt (cost 12)
  • ✅ Rate limiting on all API endpoints
  • ✅ Input validation (XSS, SQL injection prevention)
🗃 Data Security
  • ✅ PostgreSQL encrypted at rest (disk encryption)
  • ✅ SIP/RTP media encrypted via DTLS-SRTP
  • ✅ Redis authentication enabled
  • ✅ No data ever leaves internal network
👥 Access Control
  • ✅ Role-based access control (RBAC)
  • ✅ Per-conversation access control lists
  • ✅ Admin audit logs for sensitive operations
  • ✅ Session timeout & forced re-authentication

Risk Assessment

UCM6304 video bridge limit: 40 participants, 4 rooms
All-hands meetings with 40+ video participants exceed UCM6304 capacity
MEDIUM
Mitigation: Janus SFU as overflow for 40+ participants. Most team meetings (5-30 people) handled natively by UCM6304.
Single server = single point of failure
All services on one machine; hardware failure = total outage
HIGH IMPACT
Mitigation: Regular automated backups, system monitoring, future HA plan.
300+ concurrent WebSocket connections
Potential memory/CPU pressure from many active connections
MEDIUM
Mitigation: Node.js handles this natively. Redis pub/sub offloads message routing.
File storage fills up over time
Shared files, attachments, recordings accumulate
MEDIUM
Mitigation: Storage monitoring dashboard, automated cleanup policies, admin alerts.
Browser WebRTC compatibility
Some older browsers may not support WebRTC
LOW
Mitigation: All modern browsers (Chrome, Firefox, Edge, Safari) support WebRTC.

Pre-Development Checklist

UCM6304 Configuration (192.168.7.2)
  • Update firmware to 1.0.29.19 or later
  • Enable WebRTC Gateway (WSS port 8445)
  • Enable REST API access (HTTPS port 8089)
  • Generate API credentials (MD5 challenge/response)
  • Create SIP extension range (e.g., 1001–1500 of 2,000 max)
  • Set account_type: SIP(WebRTC) on extension templates
  • Configure 4 video conference rooms (40p @1080p)
  • Configure audio conference bridge (200p max)
  • Set up SSL certificate for WSS on port 8445
  • Configure SRTP (reduces to 200 concurrent calls)
  • Test SIP.js to wss://192.168.7.2:8445/ws
  • Configure call recording storage (SD/USB 3.0)
  • Enable LDAP directory
  • Verify 192.168.7.2 reachable from all employee subnets
Server & Environment Setup
  • Provision blank server (16 cores, 32GB, 1TB SSD)
  • Install Ubuntu Server 22.04 LTS
  • Assign static internal IP address
  • Install Docker & Docker Compose
  • Generate internal SSL certificates
  • Configure firewall (block all external)
  • Set up DNS entry on internal network
  • Verify connectivity to Grandstream UCM
  • Set up backup schedule
  • Create development environment
